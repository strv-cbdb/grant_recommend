# Claude.md

## 目的
このリポジトリで Claude Code を使用する際のガイドライン。

Claude Code は **既存実装との整合性を最優先**とし、不要な変更や過剰な実装を避ける。

---

## 基本方針

### 1. 既存実装に寄せる
新しい設計やルールを勝手に作らない。

作業時は以下の順で判断する。

1. 既存コードの実装を確認
2. 同様の実装パターンを探す
3. それに合わせて実装する
4. 該当がない場合のみ新規設計

NG
- 新しい命名ルールを作る
- ディレクトリ構造を変更する
- 設計を作り直す

OK
- 既存の書き方に合わせる
- 同じパターンのコードを参考にする

---

### 2. 変更範囲は最小にする
修正は **必要最小限**にする。

NG
- 関係ないコードの整形
- 大規模リファクタ
- 不要な依存追加

OK
- 対象箇所のみ修正
- 小さな差分

---

### 3. コードスタイル
以下の優先順位で合わせる。

1. リポジトリ既存コード
2. 使用言語の一般的な慣習

---

## 作業フロー

Claude Code は以下の順序で作業する。

1. 対象ファイルを確認
2. 既存実装のパターンを探す
3. 同じ書き方で実装
4. 差分を最小化

---

## 実装ルール

### 命名
命名は既存コードに合わせる。

NG
UserManager  
UserHandler  
UserLogic  

OK
既存コードと同じ命名パターン

---

### コメント
コメントは必要な場合のみ書く。

NG
# ユーザーを取得  
user = get_user()

OK
# 外部サービスの仕様によりnullが返る可能性あり

---

## Claude Code 節約TIP

### 1. 必要なファイルだけ読む
NG  
リポジトリ全体を解析  

OK  
対象ファイルのみ読む  

---

### 2. 説明は最小限
NG  
長い設計説明  

OK  
必要なコードのみ出力  

---

### 3. 既存コードを再利用
NG  
似た処理を新規作成  

OK  
既存関数を利用  

---

### 4. 小さい単位で変更
NG  
大きな変更を一度に行う  

OK  
小さい差分で変更  

---

## Claude Code への依頼方法

Claude Code に依頼する場合は以下を明確にする。

1. 目的  
2. 対象ファイル  
3. 修正内容  
4. 制約  

例

目的  
ログ出力を追加  

対象  
src/service/user_service.py  

内容  
ユーザー作成時にログを出す  

制約  
既存実装に合わせる  

---

## 禁止事項

Claude Code は以下を行わない。

- 既存設計の大幅変更
- 不要なリファクタ
- 未使用ライブラリ追加
- 全体フォーマット変更

---

## 判断基準

迷った場合は以下を優先する。

既存コード > 言語標準 > 一般ベストプラクティス
